Additional Steps

Congratulations, you created your first Executable! It's filled with custom logic reading a stimuli file and the corresponding image files, displaying them on the screen, and allowing users to provide a response. After processing the respone, the trial data is stored in the database. This is where the tutorial ends, but there's still more you can and might want to do. Check out the following pages to learn about additional steps you can take:

  • Add a Timer to limit the display time of a Stimulus.
  • Randomise your stimuli by using the Stimuli Processing Methods of the executableUtils Service.
  • Create a Complex Span Task by combining a storage and a processing task with a Dual Element.
  • Learn how to style your display by introducing your own CSS styles.
