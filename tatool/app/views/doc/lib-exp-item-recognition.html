Experimental Tasks

Item Recognition

In the Item Recognition paradigm, participants have to memorise a list of stimuli (e.g., words or faces). In the subsequent recognition phase, the previously presented items are intermixed with new items. The task is to indicate which items were presented previously and which were not. The executable allows for presenting a text or image to signal the transition from the memorisation to the recognition phase of the task.

Executable

Use the following settings in the Module Editor for this type of task:

Project tatool
Executable Type tatoolItemRecognition

Properties

The following Properties can be provided in the Module Editor to change the behaviour of the task:

Property Name Property Type Default Mandatory Description
stimuliFile Resource - Yes A stimuli file expecting stimuli according to the Tatool Stimulus definition.
stimuliPath Path - Yes A stimuli path where to load any resources from (stimuli images, input key images).
showKeys Boolean true No Shows or hides specified input keys.
encodingDuration String 3000 No Display duration for each memorandum.
blankDuration String 250 No Interval duration between memoranda.
instructionDuration String 4000 No Display duration of the instruction between the memorisation and the recognition phase.

Stimuli File

This task requires a stimuli file to run. The stimuli file has to contain the columns specified in the Stimuli File Structure below.

Available Stimuli Files

Task Project Type File Name Description
Picture Recognition uzh-luco-battery stimuli picRecognition.csv Memoranda are 15 difficult to verbalise pictures that are intermixed with 15 new pictures during the recognition phase.
Picture Recognition Practice uzh-luco-battery stimuli picRecognition_practice.csv 3 practice pictures that are intermixed with 3 new pictures during the recognition phase.
German Word Recognition uzh-luco-battery stimuli de_luco_linguistic_wordRecognition.csv Memoranda are 30 German nouns that are either concrete or abstract and intermixed with 30 new words during the recognition phase.
German Word Recognition Practice uzh-luco-battery stimuli de_luco_linguistic_wordRecognition_practice.csv 3 practice German nouns that are either concrete or abstract and intermixed with 3 new words during the recognition phase.

Stimuli File Structure

Column Name Column Values Mandatory Column Description
phase (String) Yes Determines the phase of the task.
0: Instruction, can be used to present a text or image during the transition from the memorisation to the recognition phase.
1: Memorisation.
2: Recognition.
stimulusValue[n] (String)/(Image Filename) Yes Value to display on screen. Can either be a text or an image.
stimulusValueType text/image/circle/square Yes The stimulus value type for Tatool to know how to display it.
correctResponse (String) Yes The correct response for this probe.
keyCount (String) No Number of keys to be added for this stimuli (by default 2; mandatory for more than 2 keys).
keyCode[n] (KeyCode) Yes The correct keyCode to press from the list of available KeyCodes (see Tatool Input Service).
keyLabel[n] (String)/(Image Filename) No An alternative label to display on the input key. Can either be text or an image filename.
keyLabelType[n] text/image No The internal Tatool key label type for Tatool to know how to display it.
response[n] (String) Yes Response value assigned to this key.
stimulusValueColor No Defines the colour of a text/circle/square stimulus. Can be set via color name (e.g., red) or via hexadecimal color codes (e.g., #CD0000).
stimulusType No Condition of this stimulus.
probeType No Type of this probe.
