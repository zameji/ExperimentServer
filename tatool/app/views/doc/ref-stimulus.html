Stimulus

A Stimulus is represented as a simple JavaScript object in Tatool. To use your stimuli objects with some of the services offered by Tatool (e.g., Tatool Stimulus or Tatool Grid) as they are, you have to use the naming conventions below for your properties. Following these guidelines also makes sure that someone else can quickly understand the structure of your stimuli files.

stimulusValue [String] The value to be displayed depending on stimulusValueType.
stimulusAudioValue [String] Should contain audio filename in combination with stimulusValueType of audio-text/audio-image.
stimulusType [String] Groups stimuli into a specific category or condition (e.g., congruent, incongruent, neutral).
stimulusClass [String] A CSS class name that can be applied to a stimulus.
stimulusValueColor [Color name/Color hex] A generic field used to set the color of a text stimulus.
correctResponse [String] The correct response to a stimulus that can be compared to the given response.
gridPosition [Integer] Indicates the position of a stimulus inside a Tatool Grid.
keyCode [KeyCode] Indicates the key code associated with the correct response for this stimulus.
keyLabel [String] An alternative label to display on the input key. Can be text or an image filename.
keyLabelType text/image The internal Tatool key label type for Tatool to know how to display it.
