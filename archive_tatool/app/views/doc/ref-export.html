Data Export

Tatool allows you to configure your Module with different types of Exporters suitable for different scenarios. An Exporter extracts the underlying trial data of a Module and creates a CSV file.

  • Exporter Types
  • CSV File Structure

Exporter Types

A Module can have multiple Exporters defined at the same time but only one of each type. The following Exporters are supported by Tatool:

  • Upload: Automatically exports data and uploads it to the Tatool Server running the Module after every Session.
  • Download: Allows the user to download all data to his local disk. (Currently not supported with Safari.)

CSV File Structure

The structure of the CSV file follows always the same structure: every Trial you save in your Executable corresponds to a line, and every property you add to a Trial corresponds to a column in the CSV. The following attributes are added automatically to your Trials and are therefore always the first columns of the export:

  • userCode: The internal unique and anonymised Tatool user code which allows you to identify the Trials of a specific user.
  • extId: An external identifier that can be provided when users run your experiment from a module URL.
  • moduleId: The module generating the Trials. Corresponds to the Module Label provided in the Module Editor.
  • sessionId: The internal Tatool sequence for Sessions which allows you to identify Trials belonging to the same Session.
  • sessionToken: The internal session token is a unique code assigned at execution time of a new session. When running a module from the URL this token is displayed to the user at the end and serves as proof that the user completed the experiment.
  • trialId: The internal Tatool sequence for Trials.
  • executableId: The name of the Executable generating the Trials.

If an Executable or Handler stores Module Properties, they will be available in the subsequent columns with the prefix module. For example, a Module Property with the name lastValue as part of the Executable digitTask will receive a column header module.digitTask.lastValue.

If an Executable or Handler stores Session Properties, they will be available in the subsequent columns with the prefix session. For example, a Session Property with the name lastValue as part of the Executable digitTask will receive a column header session.digitTask.lastValue.
Tatool stores one mandatory session property with the column header session.complete which indicates whether the Session was completed successfully (1, i.e., all trials were executed or the Module was stopped by pressing Escape) or not completed due to an error (0, e.g., termination of a running Session by refreshing the page or closing the browser window/tab).

All the subsequent columns are customised according to the properties you define in your Trial object. The column header naming will be [executableName].[propertyName]. For example an executable property named setSize as part of an Executable digitTask will receive a column header digitTask.setSize.
