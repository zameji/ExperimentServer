Administration

It's important to familiarise yourself with the functionality offered by the Administrator role, as this will help you to manage Users and Projects of your local Tatool instance whenever you need to.

The first time Tatool is started, a default Admin user will be created to allow you to login and setup your own users (see Getting Started for more details). You can safely delete the default Admin user as long as you provide another user with the Admin role. If you end up deleting all Admin users, simply restart the server and the default Admin user will be created again.

User Management

To use Tatool, you'll need a user name and password assigned to you. When running Tatool in lab mode, there is no user registration feature, and the only person allowed to register users is the admin user (admin@tatool-web.com by default). When logged in as an admin, you'll have access to the Admin tab in the top navigation which will lead you to the User Management screen.

On the User Management screen you can add, edit or delete users. Most importantly, you can assign roles to all users. Tatool distinguishes the following roles:

  • Modules: Access to the Modules tab allowing for installing and running Modules. Every user should have this role assigned.
  • Editor: Access to the Editor tab allowing for creating and publishing Modules.
  • Analytics: Access to the Analytics tab allowing for monitoring usage of published Modules.
  • Admin: Access to the Admin tab allowing for managing Users and Projects.

Project Management

A Project groups all the files needed for a specific experiment into one folder. For example, every Executable is part of a Project, and every stimuli file or stimulus you use is part of a Project. A special type of a Project is the External Resource, which does precisely what it says: it represents a resource from an external source (e.g., some http address) instead of a local Tatool Project.

Admins can add, edit, and delete Projects. A Project is defined by the following attributes (note that the Name, Access and Owner property can't be changed once the Project is created):

  • Name: A unique name which will represent this Project (i.e., the folder name)
  • Access: A Project can either be Public, which means every developer can access its resources, or it can be Private, meaning only the owner of the Project can access its resources.
  • Owner: The owner of a Project is the login email of the user owning that Project.
  • Description: A short description of the Project. This will be displayed in the Module Editor whenever you select a Project.
  • Executables (JSON): If you're planning on having your own Executables as part of a Project, you can provide their details as a JSON String. This allows you to select the Executables from a dropdown list in the Module Editor and auto adds the provided custom Properties. The JSON string is an array of objects, in which every object has the property customType identifying the Executable Service name and a short description which is visible in the Module Editor. Additionally, you can provide the property customProperties which is an array of objects with the properties propertyName and propertyType each.
    [ 
  { 
    "customType" : "myExecutable", 
    "description" : "A short description of the Executable", 
    "customProperties": [
      { "propertyName" : "pathProperty", "propertyType": "Path" },
      { "propertyName" : "resourceProperty", "propertyType": "Resource" }
      { "propertyName" : "booleanProperty", "propertyType": "Boolean" }
      { "propertyName" : "arrayProperty", "propertyType": "Array" }
      { "propertyName" : "arrayResourceProperty", "propertyType": "ArrayResource" }
      { "propertyName" : "stringProperty" },
    ] 
  }, 
  { 
    "customType" : "myOtherExecutable", 
    "description" : "A short description of the Executable" 
  } 
]

In order to physically add resources (e.g. Executables, stimuli files) to a specific Project, you'll have to open the tatool-web folder on the server running your Tatool instance. Within that folder you'll find the corresponding Project folders under /app/projects/*. Every Project folder comprises 4 mandatory sub-folders that contain specific types of files:

  • executables: Tatool will search for your custom Executables within this folder.
  • instructions: The Tatool Instruction Executable will search for instruction html files within this folder.
  • modules: A folder for you to keep any Module files belonging to a specific Project.
  • stimuli: A folder to keep any stimuli files provided by a Project.
