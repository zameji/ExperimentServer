Language Use and Cognition (UZH-LuCo)

Working Memory

This set of tasks measures working memory with the list-switching paradigm modelled after Oberauer, Souza, Druey, & Gade (2013) (see also Gade, Souza, Druey, & Oberauer, 2017; Oberauer, 2005). Two lists of items are presented that have to be retained in working memory. After memorisation of the lists, a series of probes is presented. The task is to indicate for each probe whether it was presented at that particular position in that particular list. In the Figural List-Switching Task, stimuli are filled or transparent simple shapes. In the Numerical List-Switching Task, stimuli are numbers which additionally have to be continuously updated according to simple arithmetic operations.

In the published version of the test battery, the tasks follow a sandwich design with 3 blocks each. In the first single-list block, each of the lists is presented separately, followed by a mixed-lists block in which the two lists switch unpredictably. After the mixed-lists block, another single-list block is completed.

Figural List-Switching Task

Executable

Use the following settings in the Module Editor for these tasks:

Project uzh-luco-battery
Executable Type lucoListSwitching

Properties

The following Properties can be provided in the Module Editor to change the behaviour of the task (note that the default value is the one used in the publication):

Property Name Property Type Default Mandatory Description
stimuliFile Resource None, see Stimuli Files below for options Yes A stimuli file expecting stimuli according to the Tatool Stimulus definition.
stimuliPath Path None, use the following:
Project: uzh-luco-battery
Type: stimuli		 Yes A stimuli path where to load any resources from (stimuli images, input key images).
showKeys Boolean true No Shows or hides specified input keys.
warningDuration String 1000 No Timer duration in ms to display a cue indicating the encoding phase.
encodingDuration String 2400 No Timer duration in ms to display the memoranda.
cueDuration String 150 No Timer duration in ms to display a cue indicating the next probe's position.
blankDuration String 250 No Timer duration in ms to wait and present a blank screen before the next probe appears.
