Executive Functions (UZH-EF)

Shifting

This set of tasks measures the mixing of and switching between mental task-sets. In all of these tasks, stimuli have to be classified according to specific rule sets. A visual cue indicates which rule has to be applied.

In the Shape/Color task, simple geometrical shapes have to be classified regarding their shape (round or angular) or color (green or blue). The total of 64 Stimuli comprises 16 round and 16 angular shapes, with each shape being used once in green and once in blue.

In the Animacy/Size task, line-drawings of simple objects or animals have to be classified regarding their animacy (living or non-living) or size (smaller or larger than a soccer ball). Line-drawings are 32 animals and 32 simple objects retrieved from the International Picture Naming Project, with half of the animals and objects being smaller (e.g., a worm or a comb), and the other half of it being larger than a soccer ball (e.g., a giraffe or a boat).

In the Parity/Magnitude task, digits (1-9 excluding 5) have to be classified regarding their parity (even or odd) or magnitude (smaller or greater than five).

In the published version of the test battery, the tasks are presented in a sandwich design each comprising 5 blocks. In the first two single-task blocks, each of the rules has to be applied separately, followed by a mixed-tasks block in which the two rules switch unpredictably. After the mixed-tasks block, the two single-task blocks are completed in reverse order.

Module

The following modules comprise the 5 blocks in the sandwich design described above for each of the three types of shifting tasks:

  • Shape/Color Task: ef_shifting_figural.json
  • Animacy/Size Task: ef_shifting_category.json
  • Parity/Magnitude Task: ef_shifting_numerical.json

    • Shape/Color Task

    Executable

    Use the following settings in the Module Editor for these tasks:

    Project tatool
    Executable Type tatoolShifting

    Properties

    See Shifting Task in the Task Library for a description of the Properties that can be provided in the Module Editor to change the behaviour of the task.

    The following values for the Properties were used in the publication:

    Property Name Value
    stimuliFile See Stimuli Files below
    stimuliPath Project: uzh-ef-battery
    Type: stimuli
    showKeys Yes
    timerEnabled No
    displayDuration N/A (not set)
    displayCueDuration 150
    randomisation N/A (not set)
