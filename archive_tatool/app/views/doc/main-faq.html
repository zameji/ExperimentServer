Q: What browsers are supported by Tatool?
A: Tatool was successfully tested on Chrome 38.x, Firefox 33.x, Internet Explorer 11, Safari 7.1 and iOS Safari 8.1. Tatool should run successfully on any modern web browser (e.g., IE 10+, FF 31+, Chrome 36+, Safari 7.1+, iOS Safari 8+) that supports ECMAScript 5 and IndexedDB. We recommend to use the latest Chrome browser due to its best overall performance (e.g., animation smoothness, timing accuracy).

Q: Does Tatool also run offline or in my lab?
A: Yes, you don't have to use tatool-web.com for your experiment. You can simply download the latest release from our GitHub repository and follow the Getting Started guide to run your own installation of Tatool in your lab.

Q: Can I use Tatool with Amazon Mechanical Turk?
A: Yes you can! Tatool Web allows you to publish your experiment via a public link which can be used as an external survey task in Amazon Mechanical Turk. Detailed instructions can be found here.

Q: Does Tatool run on an Apple iPad?
A: Yes, but iOS Safari 8+ is required. Make sure you're not running Safari in private mode.

Q: How 'good' is Tatool's timing?
A: Tatool uses the JavaScript high resolution timing API (performance.now()) when supported by the browser. This allows for sub-millisecond precision. In the case of a browser not supporting the high resolution timing API, Tatool falls back to Date.now() which allows for milisecond precision.
The accuracy of timing results depends on several other factors that need to be addressed outside of Tatool. For a good explanation on what factors influence your timing accuracy we recommend reading this (e.g., refresh rate, input device, operating system, browser) and to take appropriate measures to improve timing accuracy.

Q: Can I run my custom Executables on tatool-web.com?
A: If you're willing to make your Executables available for other researchers, we'll be happy to host your Executables for free. Otherwise we offer hosting for a small donation which helps us to pay for our fixed hosting costs. Just drop us a line.

Q: How can I use my own Stimuli with tatool-web.com?
A: If you want to use your own stimuli, you'll have to make them available somewhere where you can access them directly with an URL. If you don't have your own webhost, you can follow this tutorial to setup a free account with Cloudinary.

Q: Can I upload my own Stimuli to tatool-web.com?
A: We currently don't support the upload of custom stimuli. If you're willing to make your stimuli available for other researchers, we'll be happy to host your stimuli for free. Otherwise we offer hosting for a small donation which helps us to pay for our fixed hosting costs. Just drop us a line.

Q: How can I get an editor role on tatool-web.com?
A: If you didn't check the developer account option during the registration process, you can drop us an email.We'll let you know as soon as your developer account is activated.
