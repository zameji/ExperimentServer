trialCountHandler

The trialCountHandler provides the values for the counter status panel and can be set for every Executable.

The trial counter will always be reset to 1 at the beginning of a new Session. During a Session, the counter will be incremented by 1 after every execution of an Executable defined as a child of the current Element, regardless of whether a Trial object is saved. In the case of a Dual Element, the counter will only increment after one complete iteration (i.e., primary, secondary, primary). The counter status panel is updated at the beginning of every new execution and should therefore show the count of the current trial.

Handler Properties

The following Properties can be provided in the Module Editor to change the behaviour of the handler:

Property Name Property Type Default Mandatory Description
trialCounterStart String 1 No The number to start counting from.
reverse Boolean false No Indicating whether the trial counter should count in reverse (true) or not (false).

Module/Session Properties

The trialCountHandler doesn't save any data and will lose all information at the end of every a session.
