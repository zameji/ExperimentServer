Experimental Tasks

Choice Reaction Time

In the Choice Reaction Time paradigm, participants have to respond to a stimulus as quickly and as accurately as possible. Typically, stimuli have to be classified into two categories (e.g., decide whether a digit is odd or even, see Parity Task in the available stimuli files below). Tatool's implementation allows you to use 1 to n response options.

Executable

Use the following settings in the Module Editor for this type of task:

Project tatool
Executable Type tatoolChoiceReaction

Properties

The following Properties can be provided in the Module Editor to change the behaviour of the task:

Property Name Property Type Default Mandatory Description
stimuliFile Resource - Yes A stimuli file expecting stimuli according to the Tatool Stimulus definition.
stimuliPath Path - Yes A stimuli path where to load any resources from (stimuli images, input key images).
showKeys Boolean true No Shows or hides specified input keys.
timerEnabled Boolean false No Enables or disables timer to display stimulus.
displayDuration String 2000 No Timer duration in ms to wait for user feedback before stopping.
randomisation String (none) No Defines the randomisation criteria. If no value is set no randomisation is applied. The following values are allowed:
full: fully randomises stimuli (no duplicates until all stimuli are shown once)
full-condition: randomise with equal proportion of stimulusType (duplicates allowed)

Stimuli File

This task requires a stimuli file to run. If you're using Tatool's randomisation feature, you only need to provide every possible combination once. By setting the number of iterations you can control the number of trials. The stimuli file has to contain the columns specified in the Stimuli File Structure below.

Available Stimuli Files

Task Project Type File Name Description
Parity tatool-stimuli stimuli choice-parity.csv Uses digits 1,2,3,4,6,7,8,9 as stimuli and asks for parity (odd/even).
Magnitude tatool-stimuli stimuli choice-magnitude.csv Uses digits 1,2,3,4,6,7,8,9 as stimuli and asks whether they're less than or greater than 5.

Stimuli File Structure

Column Name Column Values Mandatory Column Description
stimulusValue (String)/(Image Filename) Yes Value to display on screen. Can either be text or an image filename.
stimulusValueType text/image/circle/square Yes The internal Tatool stimulus value type for Tatool to know how to display it.
stimulusType (String) Yes Categorize stimulus according to custom group or condition (e.g., congruent, incongruent, neutral).
stimulusValueColor (String)/(Hex Color) No An optional colour value for text/circle/square stimulusValueType. Can either be an HTML colour string or a Hex Code starting with #.
correctResponse (String) Yes The correct response for this stimulus.
keyCode (KeyCode) Yes The correct keyCode to press from the list of available KeyCodes (see Tatool Input Service).
keyLabel (String)/(Image Filename) No An alternative label to display on the input key. Can either be text or an image filename.
keyLabelType text/image No The internal Tatool key label type for Tatool to know how to display it.
keyIndex (Number) No Defines the position of the key, starting at 0. The smaller the number, the further left the key will appear.
response (String) Yes Response value assigned to this key.
